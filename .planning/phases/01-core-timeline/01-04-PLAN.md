FILE 1: src/data/tweets.ts

This is the bridge between the build pipeline and the rendering layer. It uses Vite's `import.meta.glob` to collect all `.md` file modules at build time and exports a sorted `TweetData[]` array ready for rendering.

```typescript
import type { TweetData } from '../types/TweetData';

/**
 * Collect all tweet Markdown modules from content/tweets/ at build time.
 * Vite resolves these imports statically -- no filesystem access at runtime.
 * The object shape matches the Vite plugin output: { meta: TweetMeta, html: string }.
 */
const modules = import.meta.glob<{
  meta: { mood: string; created: string };
  html: string;
}>('/content/tweets/**/*.md', { eager: true });

/**
 * All tweets, sorted reverse-chronologically (newest first).
 * Slug is derived from the filename (last path segment, minus .md extension).
 */
export const tweets: TweetData[] = Object.entries(modules)
  .map(([path, mod]) => {
    // Derive slug from filename: "content/tweets/2026/05/hello-world.md" -> "hello-world"
    const slug = path.split('/').pop()!.replace(/\.md$/, '');
    return {
      meta: {
        mood: mod.meta.mood,
        created: mod.meta.created,
      },
      html: mod.html,
      slug,
    };
  })
  .sort((a, b) =>
    new Date(b.meta.created).getTime() - new Date(a.meta.created).getTime()
  );
```

KEY ARCHITECTURAL DECISIONS:
- `import.meta.glob` with `{ eager: true }` makes all imports synchronous and bundle-time. The glob pattern `/content/tweets/**/*.md` matches the year/month directory structure.
- The glob returns an object `{ [path: string]: Module }` where each module has the shape exported by the Vite plugin: `{ meta, html }`.
- Slug derivation: `path.split('/').pop()!.replace(/\.md$/, '')` extracts the filename without extension. The `!` assertion is safe because the glob always returns file paths with filenames.
- Sort: `new Date(b.meta.created).getTime() - new Date(a.meta.created).getTime()` = descending (newest first). Uses ISO 8601 timestamps from frontmatter (which include timezone offsets per technical debt avoidance in PITFALLS.md).
- Type assertion on `modules` is necessary because `import.meta.glob` returns `Record<string, unknown>` by default. The generic parameter tells TypeScript the module shape matches our markdown.d.ts declaration.

FILE 2: src/utils/time.ts

Client-side relative time formatting using dayjs with Chinese locale. This runs in the browser (NOT at build time) to ensure timestamps reflect the viewer's current time.

```typescript
import dayjs from 'dayjs';
import relativeTime from 'dayjs/plugin/relativeTime';
import 'dayjs/locale/zh-cn';

// Configure dayjs with Chinese locale and relativeTime plugin
dayjs.extend(relativeTime);
dayjs.locale('zh-cn');

/**
 * Format an ISO 8601 timestamp as a Chinese relative time string.
 * Examples: "几秒前", "3分钟前", "2小时前", "5天前", "1个月前"
 *
 * This MUST run in the browser (client-side), never at build time.
 * At build time, Date.now() differs from page-load time,
 * causing stale relative timestamps (Pitfall 2).
 *
 * For timestamps less than 30 seconds ago, returns "刚刚" instead of "几秒前"
 * to avoid the "0秒前" appearance (UX Pitfalls in PITFALLS.md).
 */
export function formatRelativeTime(isoString: string): string {
  const date = dayjs(isoString);
  if (!date.isValid()) {
    return isoString; // Fallback: show raw ISO string for invalid dates
  }

  const now = dayjs();
  const diffSeconds = now.diff(date, 'second');

  // For very recent tweets (< 30 seconds), show "刚刚" (just now)
  if (diffSeconds < 30) {
    return '刚刚';
  }

  return date.fromNow();
}
```

DAYJS CONFIGURATION:
- `relativeTime` plugin: adds `.fromNow()` method. The plugin is included in the main `dayjs` package since v1.11.x -- no separate npm install needed.
- `zh-cn` locale: imports `'dayjs/locale/zh-cn'` which registers itself on import. `.locale('zh-cn')` sets it as the default.
- "刚刚" (just now) for < 30 seconds: Per PITFALLS.md UX Pitfalls -- avoids the "0秒前" bug.
- Invalid date fallback: returns the raw ISO string. This handles edge cases where frontmatter `created` is malformed (e.g., `created: not-a-date` from a partially corrupted file). The user sees the raw date string rather than a crash.

CRITICAL PITFALL AVOIDANCE (Pitfall 2):
- `formatRelativeTime` is called when the browser executes main.ts (DOMContentLoaded or module load), NOT at build time.
- The ISO timestamps come from frontmatter (static, build-time value) but the relative computation happens client-side.
- This avoids the hydration mismatch anti-pattern: the server never renders relative times, only ISO strings in `<time datetime="...">` elements. The client-side JavaScript replaces them with relative times.

Do NOT: compute relative time at build time. Do NOT store "3 hours ago" in the HTML output.

FILE 1: src/components/TweetCard.ts

DOM factory function: creates a single tweet card `<article>` element from TweetData. This is a pure function -- no side effects, no state, just DOM creation.

```typescript
import type { TweetData } from '../types/TweetData';
import { formatRelativeTime } from '../utils/time';

/**
 * Create a single tweet card DOM element.
 * Implements D-02: top bar with username "Sy", mood emoji, and relative time in a single row.
 * The Markdown body is rendered via innerHTML (safe -- content is pre-sanitized at build time).
 *
 * @returns HTMLElement -- the tweet card article element, ready to append to the DOM.
 */
export function createTweetCard(tweet: TweetData): HTMLElement {
  const article = document.createElement('article');
  article.className = 'tweet-card';
  article.id = `tweet-${tweet.slug}`;

  // --- Top bar: username + mood + relative time (D-02) ---
  const header = document.createElement('header');
  header.className = 'tweet-header';

  // Username (always "Sy" per PROJECT.md and TIM-01)
  const username = document.createElement('span');
  username.className = 'tweet-username';
  username.textContent = 'Sy';

  // Mood emoji (from frontmatter, displayed as content not as UI icon)
  const mood = document.createElement('span');
  mood.className = 'tweet-mood';
  mood.textContent = tweet.meta.mood;
  mood.setAttribute('aria-label', `Mood: ${tweet.meta.mood}`);

  // Relative time (computed client-side, displays ISO as datetime attribute)
  const time = document.createElement('time');
  time.className = 'tweet-time';
  time.setAttribute('datetime', tweet.meta.created);
  time.textContent = formatRelativeTime(tweet.meta.created);

  header.appendChild(username);
  header.appendChild(mood);
  header.appendChild(time);

  // --- Body: pre-rendered Markdown HTML (TIM-02) ---
  const body = document.createElement('div');
  body.className = 'tweet-body';
  body.innerHTML = tweet.html;

  // Security assertion: tweet.html was sanitized at build time by renderMarkdown()
  // (strips <script>, <iframe>, on* handlers). CSP meta tag provides defense-in-depth.

  // --- Assemble ---
  article.appendChild(header);
  article.appendChild(body);

  return article;
}
```

DESIGN DECISIONS:
- `textContent` is used for username, mood, and time -- never innerHTML for user-controlled text. This prevents XSS from frontmatter values even if marked somehow passes them through.
- `body.innerHTML` is used ONLY for `tweet.html`, which was produced by the build-time `renderMarkdown()` function (which sanitizes dangerous tags). This is the single point where innerHTML is acceptable.
- The `<time>` element uses `datetime` attribute with the ISO 8601 string. Screen readers and bots read the machine-readable datetime; users see the relative time.
- `article.id = 'tweet-{slug}'` enables browser-native fragment navigation (`#tweet-hello-world` scrolls to the card).
- Mood emoji has `aria-label` for screen reader accessibility.

FILE 2: src/components/Timeline.ts

Container component: iterates the sorted TweetData array and appends TweetCard elements to the `#timeline` container.

```typescript
import type { TweetData } from '../types/TweetData';
import { createTweetCard } from './TweetCard';

/**
 * Render the full tweet timeline into the #timeline container.
 * Handles the empty state when no tweets exist.
 *
 * @param tweets - Sorted array of tweets (newest first, from src/data/tweets.ts)
 */
export function renderTimeline(tweets: TweetData[]): void {
  const timelineEl = document.getElementById('timeline');
  if (!timelineEl) return;

  // Clear any existing content (safe to call multiple times, e.g., HMR)
  timelineEl.innerHTML = '';

  // Empty state (per UI-SPEC.md Copywriting Contract)
  if (tweets.length === 0) {
    const emptyCard = document.createElement('article');
    emptyCard.className = 'tweet-card';

    const heading = document.createElement('h2');
    heading.textContent = '还没有推文。';
    heading.style.cssText = 'font-size:18px;font-weight:600;margin-bottom:8px;color:var(--text-primary)';

    const body = document.createElement('p');
    body.textContent = '在 content/tweets/ 目录下创建你的第一篇 Markdown 推文吧。';
    body.style.cssText = 'font-size:16px;color:var(--text-secondary);margin:0';

    emptyCard.appendChild(heading);
    emptyCard.appendChild(body);
    timelineEl.appendChild(emptyCard);
    return;
  }

  // Render each tweet card in order (already sorted by tweets.ts)
  for (const tweet of tweets) {
    const card = createTweetCard(tweet);
    timelineEl.appendChild(card);
  }
}
```

DESIGN DECISIONS:
- Empty state: Per UI-SPEC.md Copywriting Contract, when `content/tweets/` contains zero valid `.md` files, display a single centered card with heading "还没有推文。" and body "在 content/tweets/ 目录下创建你的第一篇 Markdown 推文吧。" in secondary text.
- `timelineEl.innerHTML = ''` at the start ensures the component is idempotent -- safe to call during HMR or re-rendering.
- The tweets are already sorted (by tweets.ts), so the component iterates in order and appends sequentially. No additional sorting needed.
- The 1px semi-transparent divider between cards (D-08) is handled by CSS: `.tweet-card + .tweet-card { margin-top: 20px; border-top: 1px solid var(--card-divider); }` or equivalent. The component does NOT add divider elements -- CSS handles visual separation.

/**

 */
function renderFooter(): void {
  const footerEl = document.getElementById('footer');
  if (!footerEl) return;

  const text = document.createElement('p');
  text.className = 'footer-text';
  const year = new Date().getFullYear();
  text.textContent = `Powered by ViteX · ${year}`;
  footerEl.appendChild(text);

// Boot the application when the DOM is ready

  document.addEventListener('DOMContentLoaded', main);
} else {
  // DOM already loaded (script executed after page parse)
  main();

DESIGN DECISIONS:
- `import './styles.css'` tells Vite to bundle and inject the stylesheet. Without this import, Vite does not include styles.css in the output. This is the standard Vite CSS import pattern.

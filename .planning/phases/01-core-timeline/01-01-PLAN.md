Dark mode flash prevention (PROF-05, Pitfall 3): The blocking inline `<script>` in `<head>` executes synchronously before any CSS or body content renders. It reads the system `prefers-color-scheme` media query and sets `data-theme="dark"` or `data-theme="light"` on `<html>`. Pico.css uses this attribute to apply the correct theme from the very first paint. No `localStorage` check -- this project uses system preference only per user decision (no manual toggle).

Content-Security-Policy: The `<meta>` tag restricts:
- default-src 'self' -- only load resources from same origin
- style-src 'self' 'unsafe-inline' https://fonts.googleapis.com -- needed for Google Fonts CSS + inline styles from components
- font-src 'self' https://fonts.gstatic.com -- needed for Google Fonts font files
- img-src 'self' https: data: -- needed for external avatar (api.ddnsy.fun) and Bing background (bing.img.run)
- script-src 'self' 'unsafe-inline' -- needed for the dark mode blocking script
- connect-src 'self' -- no external API calls in Phase 1

CSP rationale: This is a display-only static site with no user input. A strict CSP is easy to define and prevents any injected scripts from executing -- even if a malicious Markdown post somehow bypasses marked's sanitization (defense in depth).

The `<link rel="icon">` points to `/favicon.svg` which will be created in the public directory. If favicon.svg does not exist yet, the browser simply shows no favicon -- this is acceptable.

The `<title>` is "Sy's Timeline" per the user's username "Sy" from PROJECT.md and the project's nature as a personal timeline.

Create src/types/TweetData.ts:

This file defines the core data contract for the entire application. Every component and data module depends on these interfaces. The contract is intentionally minimal -- only fields that have a defined consumer.

```typescript
/** Metadata parsed from YAML frontmatter of each tweet Markdown file. */
export interface TweetMeta {
  /** Emoji representing the author's mood when writing. e.g. "😊", "🤔", "🎉" */
  mood: string;
  /** ISO 8601 timestamp with timezone offset. e.g. "2026-05-02T10:30:00+08:00" */
  created: string;
}

/** Complete tweet data produced by the build pipeline, consumed by rendering components. */
export interface TweetData {
  /** Frontmatter metadata (mood, timestamp) */
  meta: TweetMeta;
  /** Pre-rendered HTML string from build-time marked processing. NEVER contains raw Markdown. */
  html: string;
  /** URL-safe identifier derived from the Markdown filename without .md extension. e.g. "hello-world" */
  slug: string;
}
```

Key design decisions:
- `created` is always ISO 8601 with timezone offset. Never use `Date.now()` at build time (Pitfall 2). The timezone offset (e.g., `+08:00`) is mandatory per PITFALLS.md technical debt avoidance.
- `html` is always a pre-rendered HTML string. The browser NEVER receives raw Markdown. The Vite plugin (Plan 03) handles all Markdown-to-HTML conversion at build time.
- `slug` is derived from the filename by the tweets.ts glob loader (Plan 04). It is NOT stored in frontmatter.

Create src/types/markdown.d.ts:

This ambient module declaration tells TypeScript what `.md` file imports resolve to. Without this, importing `content/tweets/**/*.md` produces a TypeScript error.

```typescript
declare module '*.md' {
  export const meta: import('./TweetData').TweetMeta;
  export const html: string;
}
```

Note: The `.md` module shape MUST match exactly what the Vite plugin (Plan 03) exports: `{ meta, html }`. If the plugin exports different names, TypeScript will catch the mismatch at compile time.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| External image URLs -> Browser | Avatar (api.ddnsy.fun) and background (bing.img.run) are third-party URLs. Browser loads them directly. |
| Markdown content -> Build pipeline | Malformed YAML or malicious Markdown could crash build or inject scripts. |
| HTML shell -> Browser | The CSP meta tag is the only defense against injected content at runtime. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Spoofing | index.html CSP | mitigate | Content-Security-Policy meta tag restricts script-src to 'self' + 'unsafe-inline' (needed for dark mode script). Blocks injected external scripts. |
| T-01-02 | Information Disclosure | External image URLs | accept | Avatar and background image URLs are in client-side code (profile.ts bundled into JS). These are public URLs by design -- no secret information. Accepting the risk as inherent to static site architecture. |
| T-01-03 | Denial of Service | Bing image URL | accept | If bing.img.run is unavailable, the CSS gradient placeholder and solid background-color fallback (Plan 02) serve as the mitigation. Accepting that the visual differentiator (Bing image) may be unavailable sometimes. |
</threat_model>
